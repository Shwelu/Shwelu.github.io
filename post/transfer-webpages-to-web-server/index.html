How to send webpages to VPS using PuTTY

Mon, Apr 17, 2017 PuTTY, sftp

Introduction

This tutorial will guide you how to serve your webpages or host a static website from a VPS with LAMP (or LEMP) stack installed. It can be considered as the next step to this tutorial or this one.

Prerequisite

  • A VPS with LAMP (LEMP) stack installed
  • username and password for an addon user with sudo user previleges for the VPS
  • The tutorial assumes that you know how to connect to the remote server using PuTTY.

Step 1: Install PuTTY

If you are on Windows, you might have used Putty to connect to your VPS. However, to send the file to the remote server, we need psftp which is a PuTTY utility used to securly transfer file to the remote server. It can be considered as CLI alternative to the FileZilla (familiar to those coming from a shared hosting environment). You can either install the psftp individually, or install the PuTTY binary with all of its utilities included. We will choose the latter for this tutorial.

Go to the page

Choose the appropriate binary (32-bit or 64-bit) according to architectue of your Windows. If you do not know which one to use, download 32-bit as 64-bit Windows supports the installation of the 32-bit apps too. Install the Putty, and make sure the psftp is on the PATH.

Transfer the files

Open cmd on Windows and move to the directory where you have saved your html files

cd C:\path\to\your\html-files

and zip your static website files (including the index.html):

C:\path\to\your\html-files>zip -r webpages.zip *

Type psftp on cmd:

C:\path\to\your\html-files>psftp

It will launch psftp prompt.

psftp: no hostname specified; use "open host.name" to connect
psftp> _

Enter user_name (for example demo) and server_IP (for example 111.111.111.111):

psftp> open demo@111.111.111.111

You will get the prompt to input the password.

Using username "demo".
demo@111.111.111.111's password:

Enter the password for the sudo user. If authentication is successful, prompt will move to the home directory of the demo user on the remote server.

Remote working directory is /home/demo
psftp> _

Transfer the zip file to the remote server

psftp> put webpages.zip

If you get the following output, your zip file has been successfully trasferred to the remote server.

local:webpages.zip => remote:/home/demo/webpages.zip
psftp>

you can see the zip file in the directory:

psftp>DIR

output will be similar to the following.

...

d----         26-Apr-17   9:21 PM            python                            
-a---         30-Apr-17   9:21 PM       1568 index.xml                         
-a---         06-May-17   2:04 PM      31896 webpages.zip
...

Move the zip file to the webroot directory

Now connect to the remote server using PuTTY

and move the zip file to the webroot directory

 mv webpages.zip /var/www/html/
 cd /var/www/html/

Change the name of the already existing index.html file

mv index.html index1.html

Now unzip the zip file

unzip webpages.zip

Test your site by typing the http://server_IP in the browser.
